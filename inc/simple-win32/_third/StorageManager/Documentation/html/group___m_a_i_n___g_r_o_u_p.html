Detailed Description

The Storage Manager Module contains all classes that are used for this library.

In short, the StorageManager was created to easily save and manage data beyond the lifespan of the program. Check the StorageManager class for further information.

Macro Definition Documentation

#define STORMAN_CRYPTED   "Valid, Hex-Encoded AES-Key (16/32 Byte)"

IMPORTANT READ DESCRIPTION.

Warning
This constant is NOT DEFINED. Normally, the StorageManager will not be able to use encryption.
If you want to use this feature, you have to include the CryptoPP-Library. The Version used is 5.6.2.

If you define this constant within your compiler, this will enable the usage of en- and decryption, and activate it. If you only use this constant, the key and iv will be generated from it. The Algorithm used is AES with EAX (Authentificated). So make shure to enter a valid AES-Key. The manager will use the HexDecoder to generate the key. If you don't want to enter the key this way, the Data can be set with StorageManager::setCrypted. But be sure to define STORMAN_NOLOAD, to prevent the Manager from failing, and call StorageManager::load manually after you set key and iv.

Remarks
A valid AES-Key in this case is a hexadecimal string, with either 32 or 64 characters (16/32 Bytes), e.g. "01234567890ABCDEF01234567890ABCDEF"
Attention
The Manager uses the SecByteBlockSink patch. Check the StorageManager.cpp file, line 56, where the patch is included. If you already have the patch, disable/remove this section

#define STORMAN_NOLOAD

IMPORTANT READ DESCRIPTION.

Warning
This constant is NOT DEFINED. Normally, the StorageManager will call StorageManager::load on it's construction. But because the Manager is located in the DataSegment, it's impossible to access it before it's created. To prevent The automatic load, simply define this constant as compiler constant. The part calling the load function will only be compiled, if this constant is not defined

Enumeration Type Documentation

enum StorageManager::LoadPolicy

Possible behaviors to load the data.

Can be used to specify the way, the StorageManager will behave on loading data via StorageManager::load
Notice that you can not affect the first load, that is done on creation of the Manager

Enumerator
REPLACE 

=0 Always replaces already created variables, default behaivior.

KEEP 

=1 Always keeps already created variables, loaded data will be truncated.

KEEPPOINTERS 

=2 Behaves like StorageManager::REPLACE, but only if type and size are the same.Will not change the variables address.
