This class is baisically a wrapper class for pointers created on Storage. It can be used ta save "pointers" beyond the execution of the program.
Usage: Create a normale pointer on the Storage, using e.g. StorageManager::getVariable. Then create an object of this class with the created pointers name as parameter. From now on you can use the StorageVariable to access this pointer. After the execution of a Program, Pointers will be invalid, so storing them is useless and can cause the program to crash. A StorageVariable, however, can be stored without any problems. You can use a StorageVariable inside another class, that was created in the storage, to basically store the data of the pointer referred by the StorageVariable

The StorageManager is a static class, used to manage Data created on the "Storage". The Storage is used as a synonym for data, that should be placed on the Harddrive.
Of course, the storage is not directly allocted on the Harddrive, because this would cause massive slow-down Effekts. Als long as the Program is running, the data will be stored at the heap. The StorageManager has it's own functions to declare and Manage Variables, that shoudl be placed on the Harddrive, and is optimized to do that. it also provides the StorageVariable class, which allows easy access to the Storage. Notice that once you added the StorageManager.cpp file to you project, an StorageManager will automatically created. The Manager is placed at the DataSegment, so it will be constructed before the main starts and deleted after it ends. if not specified different, it will load Data stored on the Harddrive on construction and save it on destruction. You can use a special Macro or the Memberfunction StorageManager::setAutoLoad to modify these.
With the StorageManager you are able to allocate and free memory, just with standard C++ functions. The main differenc is, that if you don't delete them before the program ends, the Manager will take care of them, meaning it will save them permanently on the Harddrive and then free the allocated Memory. And next time you start your program, it will automaticaly load them.

Warning
The Manager will create an XML-File, to store the data, but it will not obay the standard-xml-rules. Check StorageManager::load for further information
